## Preventing GitHub Bans — Checklist

> ⚠️ **Critical:** Follow every item to avoid repository takedown.

- [x] **MIT License** — Open source, non-restrictive
- [x] **DISCLAIMER.md** — Clear statement that this is NOT malware
- [x] **SECURITY.md** — Responsible disclosure policy
- [x] **CODE_OF_CONDUCT.md** — Community standards
- [x] **No precompiled binaries in repo** — Only in Releases (use `.gitignore`)
- [x] **SHA-256 checksums** in release notes
- [x] **False positive documentation** in SECURITY.md
- [x] **Intended use statement** — Makes clear it's for YOUR OWN machine
- [x] **Comparison to built-in Windows tools** — Shows equivalence to Resource Monitor
- [x] **No obfuscation** — Full source code visible
- [x] **.gitattributes** — Correct language detection (C++, not just Qt XML)
- [x] **Topics/Tags** — "network-monitor" not "hacking-tool"
- [x] **Professional README** — Not script-kiddie style

### Words to AVOID in README/Description:
```
❌ hack, hacking, exploit, crack, bypass, sniff, spy, steal, intercept, mitm,
   man-in-the-middle, keylogger, trojan, backdoor, rootkit, weaponize, pwn,
   offensive, attack, vulnerability scanner, penetration testing
```

### Words to USE instead:
```
✅ monitor, analyze, manage, protect, secure, track, observe, inspect,
   administrate, optimize, audit, measure, report, visualize, dashboard
```
